## Overview

Pickles Direct is a mobile app that lets industrial asset vendors self-service list equipment for auction directly from their own sites, without transporting assets to Pickles branches.

**Core user flow (SOW v1.4):**

```
Sign In / Register
    вҶ“
Terms & Conditions (first use / version bump)
    вҶ“
"How many items?" (Step 1a routing screen)
    в”ңв”Җв”Җ 1 item  вҶ’ Long Form (Asset Category вҶ’ Asset Form вҶ’ Photo Capture вҶ’ Submit)
    в””в”Җв”Җ 2+ items вҶ’ Bulk Lead Capture вҶ’ Confirmation
```

**Returning users** (session restored) skip the routing screen and land directly on Dashboard.

## Architecture Pattern

**Clean Architecture + feature-based DDD.**

Each feature is divided into three vertical layers:

| Layer | Responsibility | May import |
|-------|---------------|-----------|
| `domain/` | Entities, repository interfaces, use cases, domain failures | Nothing outside `core/error` |
| `data/` | Repository implementations, remote/local data sources, mappers | `domain/`, `core/network`, `core/storage` |
| `presentation/` | Bloc, Pages, Widgets | `domain/` use cases, `core/di`, `core/router` |

**Rules:**
- Domain layer has zero Flutter dependencies.
- Repositories return `Either<Failure, T>` (dartz) вҖ” never throw from domain.
- Use cases wrap a single repository method or compose multiple; used by Blocs.
- Pages never call repositories directly; always go through a Bloc.

## State Management

**Package:** `flutter_bloc` v9 + `bloc` v9

### Bloc conventions

- One Bloc per feature screen (or per bounded context if shared across screens).
- Events are `final class` extending a sealed `*Event` base.
- States are `final class` extending a sealed `*State` base.
- Both files are `part` files of the Bloc using `part of 'xxx_bloc.dart'`.
- Blocs are `@injectable` (factory) so each BlocProvider gets a fresh instance.

### Auth state bridge

`RouterNotifier` uses a routing-specific `AuthState` **enum** (unknown / unauthenticated / onboarding / needsRouting / authenticated), which is intentionally separate from the Bloc's sealed `AuthState` class. The `AuthStateStreamBridge` class maps Bloc states вҶ’ router enum values:

```
AuthBloc (sealed states)
    вҶ“  AuthStateStreamBridge.stream
RouterNotifier (enum)
    вҶ“  GoRouter.redirect
Navigation
```

## Dependency Injection

**Package:** `get_it` + `injectable`

### Registration order (`injection.dart`)

1. **Third-party singletons** (AppConfig, Logger, Connectivity, FlutterSecureStorage) вҖ” registered manually before `$initGetIt()`.
2. **`$initGetIt()`** вҖ” runs all `@injectable` / `@singleton` / `@lazySingleton` annotated classes (generated).
3. **Manual wiring** вҖ” `RouterNotifier` and `AppRouter` are registered manually after `$initGetIt()` because `RouterNotifier` depends on `AuthStateStream` (an interface resolved only after the generated DI runs `AuthStateStreamBridge`).

### Annotations used

| Annotation | When to use |
|-----------|------------|
| `@singleton` | One instance for app lifetime (services, repositories) |
| `@lazySingleton` | Singleton but created on first use |
| `@injectable` | Factory вҖ” new instance each time (Blocs) |
| `@LazySingleton(as: Interface)` | Singleton registered as its interface type |
| `@Singleton(as: Interface)` | Eager singleton registered as interface |

### After adding a new injectable class

```bash
dart run build_runner build --delete-conflicting-outputs
```

## Networking

**Package:** `dio` v5

### DIO client (`core/network/dio_client.dart`)

- Base URL injected from `AppConfig.middlewareBaseUrl` (set via `--dart-define=MIDDLEWARE_BASE_URL=...`).
- Default timeouts: 30s connect, 30s receive, 5min send (for photo uploads).
- **`AuthInterceptor`** вҖ” attaches `Authorization: Bearer <token>` from `TokenStorage`; on 401, attempts silent refresh via `AuthRefreshService`, retries once; on second 401, signs user out.
- **`RetryInterceptor`** вҖ” retries GET/HEAD/OPTIONS on network errors with exponential backoff (3 attempts max).
- **`PrettyDioLogger`** вҖ” only active in non-production (`AppConfig.isProd == false`).
- Certificate pinning: stub placeholder in `DioClient` вҖ” implement once Pickles provides TLS certificate fingerprints.

### Adding API clients (when middleware specs arrive)

1. Add `retrofit` and `retrofit_generator` to `pubspec.yaml`.
2. Create `lib/features/<feature>/data/datasources/<feature>_api.dart` with `@RestApi` annotation.
3. Run `dart run build_runner build`.
4. Register the generated client in `injection.dart` (or annotate with `@injectable`).

## Security

### Token storage (`core/security/token_storage.dart`)

- Access and refresh tokens stored in `flutter_secure_storage` (iOS Keychain / Android Keystore).
- `TokenStorage` exposes `saveTokens`, `getAccessToken`, `getRefreshToken`, `clearTokens`.
- `AuthRefreshService` is an abstract interface (to avoid circular DI); implemented by the auth data layer.

### Encryption (`core/security/encryption_service.dart`)

- AES-256-GCM via `encrypt` + `pointycastle`.
- Encryption key generated on first run and stored in Keychain/Keystore (never persisted to plain storage).
- Used to encrypt `assetPayloadEncrypted` in `SubmissionDrafts` before writing to SQLite.

## Offline Sync

**Package:** `workmanager` (background) + custom `SyncEngine`

### SyncEngine (`core/sync/sync_engine.dart`)

- Started from `main.dart` after DI initialisation.
- Listens to `NetworkInfo.onConnectivityChanged`; triggers `syncNow()` when connectivity is restored.
- Processes the `SyncQueue` table sequentially, one item at a time.
- Exponential backoff with jitter: `base * 2^attempt + random(0..4)` seconds, capped at 5 minutes.
- Max retries: `AppConstants.syncMaxRetryAttempts` (default: 5).
- Emits `SyncStatus` stream (idle / syncing / completed / failed) for UI sync badge.

### SubmissionSyncService (pending)

The `SubmissionSyncService` interface is defined in `sync_engine.dart`. Its implementation (`SubmissionSyncServiceImpl`) lives in the `asset_capture` data layer and performs the actual multipart HTTP upload to middleware. This will be built when the middleware OpenAPI spec is available.

## Key Conventions

- **Australian English** in all user-facing strings and comments (enforced by lint rule).
- **No hardcoded strings** in widgets вҖ” use constants or localisation keys.
- **No navigation in Blocs** вҖ” Blocs emit states; `BlocListener` in widgets handles navigation.
- **`Either<Failure, T>`** from `dartz` used for all repository return values; never `throw` from domain or data layers unless it's a programming error.
- **`const` constructors** wherever possible вҖ” lint enforces this.
- **Private types** must not appear in public APIs вҖ” use named classes.
- **`dart run build_runner build`** must be re-run after any annotation change.
- Feature flags checked via `getIt<FeatureFlagService>().isEnabled('flag_key')`.
